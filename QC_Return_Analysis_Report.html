🚨 CRITICAL FINDING — Returns growing at alarming rate
Return volume has increased from 766 lines in January to 1,319 lines in April — a 72% increase in just 4 months. The top 5 reasons account for 80.2% of total financial loss, confirming a highly concentrated, actionable problem set. Three salesmen (KRISHNA, SUDHAKAR, SHAMIL WEDASINGHE) alone account for SCR 2.12M — 40.1% of all losses.
⚠ OPERATIONAL ALERT — Customer cancellations dominate at 44.5%
"Not Required / Cancelled By The Customer" is the single largest reason (1,587 cases, SCR 2.35M). This indicates systematic failures in demand confirmation, LPO validation, and order booking processes. This is 100% preventable with proper controls.
⚠ SYSTEM ALERT — EDP errors cost SCR 638,537
EDP-related issues (wrong processes, wrong orders) account for SCR 638,537 (12.1%) — the 3rd largest category. This is an internal process failure requiring immediate system-level corrective action and staff retraining.

Business Controls & Process Improvements
Priority 1 — Critical
Mandatory LPO / Demand Confirmation
Problem: 44.5% of returns are customer cancellations post-delivery. Fix: Implement digital LPO confirmation 24 hours before delivery for all orders above SCR 1,000. Responsible: Sales Manager + IT. Steps: (1) Configure system block — no delivery note generated without LPO confirmation; (2) WhatsApp/SMS auto-confirmation to customer; (3) Salesman responsible for getting sign-off.
Expected Impact: 30–40% reduction in customer cancellation returns = SCR 700K–940K saved annually
Priority 1 — Critical
EDP Order Processing Controls
Problem: EDP errors cost SCR 638K (12.1%). Fix: (1) Implement 4-eye validation for all EDP entries above SCR 5,000; (2) System-level duplicate order detection; (3) Real-time error rate dashboard for EDP supervisors; (4) Retraining for all EDP staff. Responsible: IT + Operations Manager.
Expected Impact: 70% reduction in EDP errors = SCR 447K saved over 12 months
Priority 1 — Critical
Salesman Order Booking Validation
Problem: Wrong orders by sales staff = SCR 656K loss. Fix: (1) Mobile app order booking with product image confirmation; (2) Order value cap per customer per day without supervisor approval; (3) Real-time order history check before booking; (4) KRISHNA, SUDHAKAR, SHAMIL — supervised order booking for 90 days. Responsible: Sales Manager.
Expected Impact: 50% reduction in sales staff errors = SCR 328K saved
Priority 1 — Critical
High-Value Return Approval Workflow
Problem: Returns above SCR 10,000/line processed without escalation (LAR SCR 77K/line, WAGSTAFF SCR 32K/line). Fix: (1) Any return above SCR 5,000 requires Sales Manager approval before processing; (2) Returns above SCR 25,000 require GM approval; (3) Automated alert system; (4) Photo evidence mandatory. Responsible: Sales Manager + Finance.
Expected Impact: Reduces fraudulent/error returns; better financial controls
Priority 2 — Important
FEFO Stock Rotation & Expiry Monitoring
Problem: Near-expiry and expired stock returns (SCR 86K + SCR 74 lines expired). Fix: (1) Warehouse FEFO (First Expired, First Out) strictly enforced; (2) 30-day expiry alert for salesman territory; (3) Weekly near-expiry report with proactive return/transfer before expiry; (4) Driver checklist includes expiry date check before loading. Responsible: Warehouse + QC.
Expected Impact: Eliminate expiry-related returns; potential SCR 100K+ savings + product liability reduction
Priority 2 — Important
Demand Forecasting for Hotel & Resort Accounts
Problem: Hotel accounts (ALPHONSE ISLAND, KEMPINSKI, SAVOY) drive SCR 1.3M+ in returns — driven by seasonal occupancy changes. Fix: (1) Monthly forecast meetings with top 10 hotel accounts; (2) Occupancy-linked order sizing model; (3) Flexible delivery schedule tied to check-in calendar; (4) Consignment model for high-risk protein items. Responsible: Key Account Manager.
Expected Impact: 40% reduction in hotel-related returns = SCR 520K saved
Priority 2 — Important
Reason Code Standardization & Mandatory Capture
Problem: 127 returns have no reason code; multiple duplicate codes exist. Fix: (1) Consolidate reason codes to 15 standard codes; (2) System must require reason code before return approval; (3) Driver reason tagging on mobile at point of pickup; (4) Monthly reason code audit. Responsible: IT + Operations.
Expected Impact: Cleaner data enables faster root-cause identification — operational value
Priority 3 — Medium-Term
Salesman Incentive Redesign
Problem: Return rate not factored into salesman incentives — creates moral hazard. Fix: (1) Net Sales (Gross – Returns) as primary KPI; (2) Monthly return value deducted from commission; (3) "Zero Return" achievement bonus; (4) Public performance dashboard visible to all team. Responsible: HR + Sales Manager.
Expected Impact: Behavioral shift reduces preventable returns 20–30%
Priority 3 — Medium-Term
Packaging & Damage Prevention
Problem: 149 damage-related returns (SCR 67K). Fix: (1) Audit packaging for top 20 damage-prone SKUs; (2) Secondary protective packaging for glass/fragile items; (3) Vehicle loading training with photos; (4) Reject damaged stock at receiving before it reaches customers. Responsible: Warehouse + Logistics.
Expected Impact: 60% reduction in damage returns = SCR 40K+ saved
Priority 3 — Strategic
Real-Time Power BI Return Dashboard
Problem: Management currently lacks live visibility into return trends. Fix: (1) Deploy Power BI connected to live ERP data; (2) Daily automated refresh; (3) Salesman, reason, brand drill-throughs; (4) Alert triggers when thresholds breached; (5) Mobile-accessible for field managers. Responsible: IT + Analytics Team.
Expected Impact: Enables proactive management; 3–5 day faster corrective action cycle

DAX Measures — Complete Library
01 — Total Returns (Count)
Total Returns = COUNTROWS('MAIN')
02 — Total Return Value
Total Return Value = SUM('MAIN'[Returnvalue])
03 — Average Return Value
Avg Return Value = DIVIDE(
  [Total Return Value],
  [Total Returns],
  0
)
04 — Monthly Growth % (MoM)
Monthly Growth % =
VAR CurrentMonthValue = [Total Return Value]
VAR PrevMonthValue = CALCULATE(
  [Total Return Value],
  DATEADD('DateTable'[Date], -1, MONTH)
)
RETURN
DIVIDE(
  CurrentMonthValue - PrevMonthValue,
  PrevMonthValue,
  0
)
05 — Return Value Share %
Return Share % = DIVIDE(
  [Total Return Value],
  CALCULATE([Total Return Value], ALL('MAIN')),
  0
)
06 — Running Total (Cumulative Value)
Running Total Value = CALCULATE(
  [Total Return Value],
  FILTER(
    ALLSELECTED('DateTable'[Month]),
    'DateTable'[Month] <= MAX('DateTable'[Month])
  )
)
07 — Salesman Rank by Return Value
Salesman Rank = RANKX(
  ALL('MAIN'[Sm Name]),
  [Total Return Value],
  ,
  DESC,
  DENSE
)
08 — Pareto Cumulative %
Pareto Cumulative % =
VAR CurrentReason = SELECTEDVALUE('MAIN'[Reason Desc])
VAR CurrentValue = [Total Return Value]
VAR AllReasonsTable = ADDCOLUMNS(
  ALL('MAIN'[Reason Desc]),
  "ReasonValue", [Total Return Value]
)
